VPN 提示错误 711 的解决办法

发表于 2014-05-26 分类于 软件使用

今天连 VPN 时遇到了 “错误 711 无法加载远程访问服务连接管理器。”

上网查了一下，说是要启动以下三项服务:（通过开始→运行→输入 services.msc →回车）

Telephony

Remote Access Auto Connection Manager

Remote Access Connection Manager

查看了一下服务，果然如此，三项都属于未启动状态。虽然不知道怎么就停了，大概也许可能说不定又是安装过的某个软件搞的，不管。

但我一点重启，提示

1067: 依赖的服务未运行。

我擦……(；°д°)……继续查看依赖服务，发现 Secure Socket Tunneling Protocol Service 这个服务没有运行。启动之，提示：

因为依赖的服务都没有运行所以我运行了一下就停了。

我了个大去！！！

(；°д°)！
(；°д°)！！
(；°д°)！！！

卖萌不带这样的。

按网上文章的后续，把 C:\Windows\System32\LogFiles\WMI 这个文件夹 “管理员获得所有权” 就可以了。但重启系统以后依然不行。

怒了，不断地查依存和被含依存关系，把 ICS、WMI 等服务全查出来。查了一下微软文档，罗列了一下目录，发现system32 下 wbem 有重点嫌疑。同样添加管理员取得所有权以后再次重启，VPN 就可用了。

嗯，标准的程序员调试技巧。

“坏了，为什么呢？”

“还是坏的，为什么呢？”

“哎~ 好了。但是为什么呢？”

就这么记一笔吧。
